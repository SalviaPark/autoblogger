# 애드센스 승인용 콘텐츠 요구사항 문서

문서 버전: v1.0  
작성일: 2026-07-09  
용도: AI 콘텐츠 생성 시스템, 자동 발행 파이프라인, 편집 검수 기준의 공통 요구사항

## 1. 목적

이 문서는 애드센스 승인을 목표로 하는 블로그/콘텐츠 사이트에서, AI가 생성하거나 자동 발행하는 글이 충족해야 할 최소 품질 기준을 정의한다.

핵심 목표는 다음과 같다.

- 고유하고 유용한 원본 콘텐츠만 발행한다.
- 검색 유입만 노린 저가치 대량 문서를 차단한다.
- 사람이 읽었을 때 실제 도움이 되는 페이지를 만든다.
- 애드센스 및 Google Search 정책에 저촉될 가능성을 낮춘다.

## 2. 적용 범위

이 문서는 다음 대상에 적용된다.

- AI가 초안을 생성하는 모든 블로그 글
- 템플릿 기반으로 대량 생성되는 정보형 페이지
- FAQ, 비교, 가이드, 설명, 체크리스트형 문서
- 자동 발행 전 검수 로직

이 문서는 다음 대상에는 직접 적용하지 않는다.

- 순수 이미지 갤러리형 페이지
- UGC 중심 커뮤니티 게시물
- 법률, 의료, 세무, 금융 자문을 직접 제공하는 고위험 문서

## 3. 정책 해석 원칙

시스템은 다음 원칙을 기본 전제로 삼는다.

- AI 사용 자체는 금지 대상이 아니다.
- 그러나 AI를 이용해 가치 없는 유사 페이지를 대량 생산하는 것은 금지 위험이 높다.
- 글은 검색엔진용이 아니라 사람용이어야 한다.
- 다른 사이트 내용을 재포장하거나 번역 복붙한 문서는 발행하지 않는다.
- 글마다 독자적인 정보 가치가 명확해야 한다.

## 4. 콘텐츠 유형 요구사항

발행 허용 대상 글은 아래 유형 중 하나여야 한다.

- 문제 해결형 가이드: 사용자가 특정 문제를 해결하도록 돕는 글
- 비교형 문서: 둘 이상의 선택지를 비교하고 차이를 설명하는 글
- 설명형 문서: 개념, 절차, 조건, 실수 포인트를 명확히 정리한 글
- 체크리스트형 문서: 실행 전에 확인해야 할 항목을 구조화한 글
- 데이터 정리형 문서: 표, 목록, 기준, 조건을 직접 재구성해 제공하는 글

발행 비권장 유형은 다음과 같다.

- 제목만 다르고 본문 구조가 거의 같은 글
- 일반론만 반복하는 얕은 문서
- 검색어를 억지로 변형해 만든 문서
- 출처 없는 주장 나열
- 특정 지역명, 상품명, 연도만 바꿔 찍어낸 문서

## 5. 필수 품질 기준

모든 글은 아래 기준을 충족해야 한다.

### 5.1 원본성

- 글의 핵심 설명은 자체적으로 다시 구성되어야 한다.
- 외부 정보를 참고하더라도 단순 요약이나 재배열 수준에 머물면 안 된다.
- 최소 2개 이상의 독자적 요소가 포함되어야 한다.

독자적 요소 예시:

- 직접 정리한 비교표
- 단계별 판단 기준
- 실수 방지 팁
- 상황별 예외 설명
- 선택 기준 요약
- 사용자 관점 FAQ

### 5.2 유용성

- 글은 하나의 명확한 질문 또는 검색 의도를 해결해야 한다.
- 사용자가 다음 행동을 결정할 수 있을 만큼 구체적이어야 한다.
- 추상적인 조언만 나열해서는 안 된다.

### 5.3 충분한 깊이

- 주제 정의만 하고 끝나면 안 된다.
- 최소한 다음 중 4개 이상을 포함해야 한다.

- 핵심 요약
- 대상 독자 설명
- 절차 또는 판단 순서
- 비교 또는 선택 기준
- 주의사항
- 자주 하는 실수
- FAQ
- 관련 주제 내부 링크

### 5.4 구조와 가독성

- 제목은 하나의 의도를 명확히 설명해야 한다.
- 서론 2~4문장 안에 글이 해결하는 문제를 밝혀야 한다.
- 소제목 구조는 논리적이어야 한다.
- 문단 길이는 과도하게 길지 않아야 한다.
- 표, 목록, 강조 문장을 적절히 사용해야 한다.

### 5.5 신뢰성

- 사실 주장에는 근거가 있어야 한다.
- 날짜, 조건, 국가, 금액, 규정이 포함된 경우 최신성 검토가 필요하다.
- 불확실한 내용은 단정형 문장으로 쓰지 않는다.

## 6. 금지 패턴

다음 중 하나라도 해당하면 발행 금지 처리한다.

- 타 사이트 문장을 거의 그대로 재서술한 글
- 번역 후 표현만 바꾼 글
- 의미 없는 키워드 반복
- 제목과 본문 내용 불일치
- 실질 정보 없이 문장 수만 늘린 글
- 같은 템플릿을 대량 복제한 문서
- 민감 주제에서 근거 없는 단정
- 스크랩 콘텐츠, 저작권 침해 가능 콘텐츠
- 클릭 유도형 과장 표현
- 검색엔진 노출만 목적으로 만든 얕은 페이지

## 7. 권장 문서 템플릿

모든 AI 생성 글은 아래 기본 템플릿을 우선 사용한다.

1. 문제 정의
2. 핵심 답변 요약
3. 상세 설명
4. 비교 또는 판단 기준
5. 실수/주의사항
6. FAQ
7. 관련 글 링크

예시 구조:

```text
제목
도입: 이 글이 해결하는 질문
핵심 요약
왜 중요한가
어떻게 판단하면 되는가
상황별 차이
주의해야 할 점
자주 묻는 질문
다음에 읽을 글
```

## 8. 자동 생성 시스템 요구사항

AI 생성 시스템은 아래 기능을 지원해야 한다.

### 8.1 입력 요구사항

- 메인 키워드
- 검색 의도 분류
- 대상 독자
- 글 유형
- 금지 주장/금지 주제
- 반드시 포함할 독자적 요소

### 8.2 생성 요구사항

- 첫 문단에서 질문과 해결 범위를 명시한다.
- 주제에 맞는 실제 판단 기준을 포함한다.
- 독자적 요소 2개 이상을 반드시 포함한다.
- FAQ는 본문 반복이 아니라 보완 질문이어야 한다.
- 결론은 독자의 다음 행동을 제시해야 한다.

### 8.3 검수 요구사항

자동 검수 단계에서 아래 항목을 체크한다.

- 유사 문서 중복률
- 제목-본문 일치도
- 문서 깊이
- 금지 표현 포함 여부
- 민감 주제 여부
- 내부 링크 존재 여부
- 표 또는 구조화 정보 포함 여부
- 사실 검증 필요 요소 포함 여부

## 9. 선택적 승인 규칙

아래 조건을 만족하면 자동 발행 후보로 분류한다.

- 중복률 낮음
- 금지 패턴 없음
- 문서 구조 완성
- 독자적 요소 2개 이상 포함
- 검색 의도 해결 명확

아래 조건이면 사람 검토 대상으로 분류한다.

- 의료, 법률, 세무, 금융 등 민감 주제
- 최신 수치, 규정, 정책 의존 문서
- 핵심 허브 문서
- 수익 가능성이 높은 메인 키워드 문서
- 품질 점수가 애매한 문서

아래 조건이면 폐기 또는 재작성한다.

- 일반론만 있음
- 중복도가 높음
- 독자적 요소 부족
- FAQ를 제외하면 실질 내용이 부족함
- 제목만 바꾸면 다른 글과 구분이 어려움

## 10. 승인 판단 체크리스트

발행 전 아래 질문에 모두 `예`라고 답할 수 있어야 한다.

- 이 글은 특정 질문을 실제로 해결하는가?
- 이 글만의 독자적인 정보가 있는가?
- 다른 글 제목만 바꾼 복제품처럼 보이지 않는가?
- 사용자가 읽고 다음 행동을 결정할 수 있는가?
- 글 구조가 명확하고 읽기 쉬운가?
- 과장, 오해 유도, 근거 없는 단정이 없는가?
- 스크랩/번역복붙/재가공 냄새가 나지 않는가?

하나라도 `아니오`이면 자동 발행 금지다.

## 11. 최소 사이트 수준 요구사항

개별 글 품질 외에 사이트 전체도 아래 조건을 충족해야 한다.

- 사이트 주제가 명확하다.
- 카테고리 구조가 정리되어 있다.
- 중복/유사 페이지가 과도하지 않다.
- 소개, 문의, 개인정보처리방침 등 기본 신뢰 페이지가 있다.
- 광고보다 콘텐츠가 중심이다.
- 모바일에서도 읽기 쉽다.

## 12. KPI와 운영 기준

초기 승인 목표 단계에서는 양보다 품질을 우선한다.

- 하루 발행량보다 문서당 품질 점수를 우선한다.
- 유사 글 대량 확장 전 대표 문서 성과를 먼저 본다.
- 핵심 허브 문서부터 먼저 완성한다.
- 승인 전에는 저가치 페이지 수를 최소화한다.

권장 초기 운영 규모:

- 허브 문서 5~10개
- 하위 고품질 문서 30~70개
- 주제 일관성 유지

## 13. 시스템 수용 기준

시스템은 다음 기준을 만족해야 운영 투입 가능으로 본다.

- 생성 글이 기본 템플릿을 안정적으로 따른다.
- 금지 패턴을 자동 탐지할 수 있다.
- 중복 문서 생성을 억제할 수 있다.
- 민감 주제를 자동 분류할 수 있다.
- 사람 검토가 필요한 문서를 별도 큐로 보낼 수 있다.
- 발행 전 최종 품질 상태를 점수 또는 등급으로 표시할 수 있다.

## 14. 구현 메모

실제 구현 시 프롬프트 또는 룰 엔진은 다음 방향을 가져야 한다.

- "많이 써라"가 아니라 "하나를 제대로 해결하라"
- "SEO 문구를 넣어라"가 아니라 "독자가 판단할 정보를 넣어라"
- "문단을 늘려라"가 아니라 "비교, 기준, 예외를 추가하라"
- "비슷한 문서를 확장하라"가 아니라 "중복 의도를 합쳐라"

## 15. 참고 기준

이 문서는 아래 공식 기준을 바탕으로 작성되었다.

- AdSense는 고유하고 흥미로운 원본 콘텐츠를 요구한다.
- 페이지는 방문자에게 관련성이 있고 좋은 사용자 경험을 제공해야 한다.
- 스크랩 또는 저작권 침해 콘텐츠는 허용되지 않는다.
- 자동화나 생성형 AI를 사용하더라도, 사용자 가치를 더하지 않는 대량 페이지는 검색 스팸 위험이 높다.
